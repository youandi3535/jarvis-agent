# JARVIS Architecture Decision Records (ADR)

이 디렉토리는 JARVIS 의 *결정 사유* 를 보관하는 **단일 진실 소스** 다.
CLAUDE.md 는 *현재 적용 중인 규칙* 을 박제하고, ADR 은 *왜 그 규칙이 만들어졌는지* 의 *역사·근거·포기한 대안* 을 박제한다.

## ADR 이 필요한 이유

CLAUDE.md 에 *결정 사유* 를 같이 박으면:
- 규정 본문이 길어져 신규 작업자가 핵심 규칙을 놓침
- 결정 번복·재검토 시 *원래 이유* 추적이 어려움
- 시간이 지나면 *근거* 만 유효하고 *적용 형태* 는 진화해야 할 때가 있음 — 둘이 분리돼야 진화 가능

## 작성 형식

각 ADR 은 다음 섹션을 따른다:

```
# NNN. 결정 제목

## 상태
확정 / 검토 중 / 폐기 / 대체됨 (날짜)

## 배경
어떤 문제·상황에서 결정이 필요했는가

## 결정
무엇을 선택했는가

## 이유
왜 이 선택인가

## 포기한 대안
검토했다가 채택하지 않은 옵션들 + 각각의 포기 사유

## 결과
이 결정이 다른 곳에 어떻게 반영되는가 (CLAUDE.md 어느 섹션·어느 파일·어느 검증 명령)
```

## 변경 정책

- **신규 추가**: 다음 번호 (`NNN-name.md`) 로 작성. INDEX 갱신.
- **번복**: 기존 ADR 의 상태를 `폐기 (날짜)` 로 변경 + 새 ADR 작성 + 이전 ADR 에 `상태: 대체됨 by NNN-newname.md` 박제.
- **수정**: *근거·결과* 의 미세 보강은 직접 편집 가능. *결정 자체* 변경은 새 ADR 작성.

## CLAUDE.md 와의 관계

CLAUDE.md 안에서 결정의 *이유* 를 길게 설명하는 대신, 다음 형식으로 참조:

```markdown
**근거**: [ADR 003 — BLOG_SUPREME_LAW.md 14조](docs/decisions/003-blog-supreme-law.md)
```

이렇게 하면 CLAUDE.md 는 *짧고 명확한 규칙 본문* 만 유지하고, 결정 추적은 ADR 로 분리된다.
